# Agent Notes — proto-circle

Context for AI agents working on this repo. Read this first.

## What this project is

Hackathon prototype of an **Instant Attribution & Micropayments Engine**:
transparent attribution plus instant USDC micropayments for creator-driven
online traffic on **Arc Testnet** (Circle's L1 for stablecoins).

The working name for this repo is intentionally neutral — long-term product
branding is out of scope for the hackathon artifact and should not be
introduced into public-facing copy.

**Full brief**: [`docs/product-brief.md`](docs/product-brief.md) — read it if
you need scope, data model, or API surface details.

## Stack

- **Runtime**: Node 20+ with ESM (`"type": "module"` in package.json)
- **Framework**: Next.js 15 (App Router) + TypeScript — API routes and
  dashboard UI in a single project
- **Storage**: SQLite via `better-sqlite3` (file at `./data/app.db`), zero
  setup, synchronous API
- **Payment rail**: Circle Developer-Controlled Wallets on Arc Testnet via
  `@circle-fin/developer-controlled-wallets`
- **Package manager**: npm

## Key domain concepts

- **Influencer** — creator with an external Arc wallet address that receives
  payouts.
- **CampaignLink** — short code (e.g. `/r/abc123`) tied to one influencer and
  one merchant/target URL. Generates the `campaign_link_id` used in events.
- **UserEvent** — raw timestamped interaction (`click`, `landing`,
  `conversion`). Always keyed by an `anonymous_user_id` (cookie-based).
- **AttributionDecision** — result of running the engine over a user's events
  leading up to a qualifying event. Contains a human-readable `rationale`
  JSON blob for dashboard transparency.
- **AttributionShare** — per-influencer weight + USD amount derived from a
  decision. A decision may have 1 or more shares.
- **Payout** — actual USDC transfer record, one per share. Holds Circle tx ID
  and tx hash once settled.

## Attribution policy (default)

**Recency-weighted within a lookback window** (default: 7 days).

For a conversion event, find all eligible click events by the same
`anonymous_user_id` inside the window. Assign each eligible click a weight of
`1 / (age_in_hours + 1)`, normalize weights to sum to 1, split the conversion
value across influencers by summed weight.

Safeguards: dedupe exact duplicates, ignore events outside the window, always
store the full rationale.

## Circle specifics

- **Network**: Arc Testnet (`ARC-TESTNET`), USDC token address
  `0x3600000000000000000000000000000000000000`
- **Platform wallet**: one developer-controlled wallet funds all payouts.
  Address stored in `.env` as `CIRCLE_WALLET_ADDRESS`.
- **Entity secret**: registered once during `scripts/create-wallet.ts`. If
  already present in `.env`, the script must skip registration (re-registering
  will fail).
- **Block explorer**: https://testnet.arcscan.app/tx/{hash}
- **Faucet**: https://faucet.circle.com (select Arc Testnet)

## Environment variables

All loaded from `.env` (gitignored). See `.env.example` for the full list.

- `CIRCLE_API_KEY` — test API key from console.circle.com (required)
- `CIRCLE_ENTITY_SECRET` — 32-byte hex, generated by the setup script
- `CIRCLE_WALLET_ID` — UUID of the platform payout wallet
- `CIRCLE_WALLET_ADDRESS` — 0x address of the platform payout wallet
- `CIRCLE_WALLET_BLOCKCHAIN` — always `ARC-TESTNET` for this hackathon

## Conventions

- Use ESM imports; no CommonJS `require`.
- Use `.ts` extensions in imports when TS config requires it (Next.js doesn't).
- Never commit `.env`, `data/`, or `output/`.
- Keep the attribution `rationale` field serializable JSON — the dashboard
  renders it directly.
- Money amounts: strings in the Circle SDK (`amount: ["5"]`), numbers
  internally. Always store USD as number in DB, convert to string at the
  Circle boundary.
- Timestamps: ISO 8601 strings at API boundary, Unix ms internally.

## Running

```bash
npm install
npm run dev        # starts Next.js on http://localhost:3000

# One-time platform setup (after Circle API key is in .env):
npm run setup      # runs scripts/create-wallet.ts
```

## Security posture (hackathon only)

- API endpoints are NOT authenticated. Fine for a local demo, not for
  production. Add merchant tokens if this ships further.
- Testnet only. Do not reuse these credentials on mainnet.
- Entity secret is sensitive even on testnet — do not log it, do not commit it.
